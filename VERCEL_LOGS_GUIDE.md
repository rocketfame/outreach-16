# Як переглядати логи в Vercel

## Де знайти логи

### 1. Через Vercel Dashboard (веб-інтерфейс)
1. Відкрийте [Vercel Dashboard](https://vercel.com/dashboard)
2. Виберіть ваш проект (`outreach-16`)
3. Перейдіть до вкладки **"Deployments"**
4. Виберіть останній deployment (найновіший)
5. Натисніть на deployment, щоб відкрити деталі
6. Перейдіть до вкладки **"Logs"** або **"Runtime Logs"**
7. Там ви побачите всі логи з префіксом `[articles-api]`, `[injectAnchorsIntoText]`, `[blocksToHtml]` тощо

### 2. Через Vercel CLI (реальний час)
```bash
# Встановіть Vercel CLI (якщо ще не встановлено)
npm i -g vercel

# Увійдіть в Vercel
vercel login

# Переглядайте логи в реальному часі
vercel logs --follow

# Або логи конкретного deployment
vercel logs [deployment-url]
```

### 3. Через Vercel Dashboard -> Functions
1. Відкрийте проект в Vercel Dashboard
2. Перейдіть до **"Functions"** в меню
3. Виберіть функцію `/api/articles`
4. Переглядайте логи виконання функції

## Що шукати в логах

### Ключові префікси для пошуку:
- `[articles-api]` - основні логи API
- `[articles-api-debug]` - детальні debug логи
- `[injectAnchorsIntoText]` - логи інжекції плейсхолдерів
- `[blocksToHtml]` - логи конвертації блоків в HTML

### Важливі логи для перевірки:
1. **"Converting blocks to HTML"** - показує скільки плейсхолдерів знайдено в блоках
2. **"HTML after blocksToHtml"** - показує скільки плейсхолдерів залишилося після конвертації
3. **"Final HTML generated"** - показує фінальний результат (скільки лінків, скільки плейсхолдерів)
4. **"ERROR: Placeholders still present"** - помилка якщо плейсхолдери не замінилися
5. **"ERROR: No links found"** - помилка якщо лінки взагалі не згенерувалися

## Як фільтрувати логи

В Vercel Dashboard можна використовувати пошук:
- Введіть `[articles-api]` щоб побачити тільки логи API
- Введіть `ERROR` щоб побачити тільки помилки
- Введіть `placeholder` щоб побачити логи про плейсхолдери

## Якщо логи не з'являються

1. Перевірте, що deployment успішний (зелений статус)
2. Переконайтеся, що ви дивитеся правильний deployment (найновіший)
3. Спробуйте оновити сторінку або перезапустити функцію
4. Перевірте, що функція виконується (згенеруйте статтю знову)
